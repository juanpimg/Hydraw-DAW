H ydraw is what happens when you build a DAW with the same stack you’d build a game with. Native C++17 audio, a hybrid HTML UI rendered by WebKit, no Electron, no Java, no proprietary plugin formats. Drag a WAV in, arm a track, load a CLAP, hit space. The window doesn’t even breathe hard.

01 / what it does

Everything you need to lay down a track.
Nothing you don’t.

  1. 01

    Audio engine

    A miniaudio callback pumping the mix bus in real time. Lock-free atomics feed the UI at 30 Hz so meters don’t fight the render thread.

    • low-latency
    • alsa
    • pulse
  2. 02

    Hybrid UI

    Layout, transport, mixer and FX chain drawn in HTML + Tailwind. The window is GTK 3, the renderer is WebKit2GTK. 22 native bindings keep JS honest.

    • html5
    • webkit
    • cn() bridge
  3. 03

    CLAP hosting

    Load any .clap plugin you find. Per-track FX chains via dlopen. PeakEater ships in the box as a soft-clipper for the master bus.

    • clap 1.2
    • fx chains
    • bypass
  4. 04

    Drag, drop, done

    Drop WAVs from your file manager straight onto a track. Drop a folder, get a session. Drop a clap on the FX chain, it just works.

    • text/uri-list
    • waveform render
    • 16 tracks
02 / under the hood

A clean seam between pixels and samples.

Hydraw treats the UI as a thin client over a deterministic audio core. One direction of data flow, one bridge, no magic. 

// html5 · tailwind · vanilla js       (assets/index.html)
┌──────────────────────────────────────────────┐
│  transport  ·  timeline  ·  mixer  ·  fx      │
└──────────────────────────┬───────────────────┘
                           │  cn(name, arg)    <-  22 native bindings
                           ▼
// c++17 · webview/webview 0.12        (src/main.cpp)
┌──────────────────────────┴───────────────────┐
│  atomic state  ·  eval() update thread (30Hz) │
└──────────────────────────┬───────────────────┘
                           │
        ┌──────────────────┼──────────────────┐
        ▼                  ▼                  ▼
// audio engine            // clap host        // file i/o
   AudioEngine           PluginHost           WavLoader
   miniaudio callback      dlopen(.clap)         RIFF parser
   peak cache (lock-free)  per-track chains      waveform peaks
        │                  │                  │
        └──────────────────┼──────────────────┘
                           ▼
// linux kernel
   alsa  ·  pulseaudio  ·  pipewire

The audio callback never touches the webview. UI updates ride on atomics. You can read the engine in src/AudioEngine.cpp and have it in your head in an afternoon.

04 / get it running

Three commands.
Then you’re making noise.

  1. 01

    Install dependencies

    GTK 3, WebKit2GTK 4.1, ALSA. The script does the work for you.

    $ chmod +x scripts/install_deps.sh && ./scripts/install_deps.sh
  2. 02

    Build

    CMake pulls miniaudio and webview/webview 0.12.0 via FetchContent.

    $ cmake -B build && cmake --build build -j$(nproc)
  3. 03

    Launch

    1.5 MB binary. No installer, no daemon, no telemetry.

    $ ./build/hydraw_daw
05 / build it with us

Hydraw is one engineer on weekends.
It needs hands.

No NDA, no CLA, no design committee. Open a PR, push a plugin, write a test, fix a typo. The good-first-issues below are good.

engine

Audio & DSP

You like reading callback graphs. The engine is in src/AudioEngine.cpp. Recording, automation, sample-accurate transport, loop regions — all on the table.

  • C++17, lock-free atomics
  • miniaudio backend
  • PCM, float32 buffers
host

CLAP & plugins

The plugin layer (src/Plugin/) is a thin dlopen wrapper. Add GUI embedding, state save/load, parameter modulation, MIDI 2.0.

  • CLAP 1.2 spec
  • per-track FX chains
  • testing with free CLAPs
ui

Web UI & bindings

HTML + Tailwind + vanilla JS, bridged into C++ via webview. 22 bindings exist. 200 are possible. Make cn() sing.

  • HTML5, modern JS
  • no build step required
  • WebKit-friendly CSS
test

QA, docs, samples

Run the binary on weird distros, file issues with journalctl output, write a getting-started tutorial, ship a starter sample pack.

  • Arch, Fedora, Nix, Ubuntu
  • markdown & examples
  • WAV test fixtures
